Job Lander — Dual-Agent Architecture

One orchestrator, two specialist agents, three human gates. Backed by a Google Sheet (memory) and Gmail (delivery).
Agent 1 (Job Finder) Agent 2 (Resume Customizer) Human approval gate Memory / delivery
Inputs
profile.json (the only seeker-specific config) + standard résumé .docx
Job Lander — Orchestrator
The manager. Loads profile, runs the flow, enforces the 3 gates, owns memory & delivery. Delegates; never searches or writes itself.
Memory — Google Sheet
Every job logged: seen → approved → tailored → sent. Dedupes so nothing is re-recommended.
▸ Agent 1 — Job Finder
Searches SerpApi Google Jobs (LinkedIn + Indeed + Glassdoor + company sites, full JDs). Hard-filters (location, role, seniority ceiling, sponsorship, salary), scores 0–10 vs. résumé, dedupes. Returns a ranked shortlist — never pads.
→
🧑 Gate 1 — Approve
Seeker approves which shortlisted jobs to pursue. Only approved jobs continue.
→
🧑 Gate 2 — Clarify
Agent 2 asks one batched set of questions for gaps it won't guess at. Real answers, never invention.
→
▸ Agent 2 — Resume Customizer
Tailors résumé per approved job from real experience + answers. Outputs .docx + PDF + recruiter ranking.
Assemble packet→ Gmail — auto-send to seeker (résumés + apply links)→ 🧑 Gate 3 — Seeker applies (manual, always)
Claude Code — agent runtime SerpApi Google Jobs — discovery (LinkedIn/Indeed/Glassdoor) Google Workspace MCP (self-hosted) — Sheets + Gmail python-docx — résumé generation LibreOffice — PDF
Automate the exhausting middle • keep the human at every real decision • never fabricate • free to run
